
.. _Zabbix特性:
****************
3.Zabbix特性
****************

概览
====
Zabbix是一个高度集成的网络监视方案，在一个软件中提供了丰富的功能。

数据采集
--------
 * 可用性数据与性能数据采集
 * 支持SNMP（同时支持trapping与polling），IPMI，JMX.VMware 
 * 可自定义数据采集
 * 可定期采集数据
 * 服务器\代理服务器和客户端均可执行数据采集

灵活的阈值定义
--------------
 * 你可以非常灵活地从后端引用相关数据来定义一个阈值，在Zabbix里，一个阈值被称为tigger。

高度可配置的警报
------------------
 * 通知功能可以定制发送内容、接收人和通知发送媒介（比如E-mail）
 * 使用宏变量可以是有意义和有用的通知
 * 具有自动操作功能，包含远程执行命令的能力

实时绘图
--------
 * 监视items并使用内建的绘图功能实时生成图像 

Web站点监视能力
----------------
 * Zabbix在Web页面上按照路径模拟点击以测试功能性和响应时间。

丰富的可视化选项
----------------
 * 能够创建自定义图表，它可以将多个项目组合到单个图表中
 * 创建网络拓扑图
 * 自定义仪表板和侧边栏
 * 可视化报告
 * 资源高级概览（商业版提供）

历史数据存储
------------
 * 使用数据库存储数据
 * 配置历史回溯
 * 内建的数据整理功能

易于配置
-------------
 * 所有的被监视设备都被称为host
 * 从数据库中选择被监视的host
 * 可以对需要监视的设备应用模板以简化配置过程

使用模板
------------
 * 在模板中批量配置监视项目
 * 模板之间可以互相嵌套

网络发现
----------------
 * 自动发现网络设备
 * 客户端自动注册
 * 自动发现文件系统、网络接口和SNMP OIDS

快速Web接口
------------------
 * 基于PHP驱动的的Web访问接口
 * 可从任意地址访问
 * 你可以配置属于你自己的前端
 * 具有日志审计功能

Zabbix API
-----------------
 * ZabbixAPI为大规模部署提供了可编程接口，第三方也可以利用这些接口集成到Zabbix或者对Zabbix进行自定义

权限管理系统
----------------
 * 安全的用户认证系统
 * 提供了权限分级，可以界定不同用户看到不同的视图

全功能而且可扩展的客户端
-------------------------
 * 部署在被监视的机器上
 * Linux和Windows平台均可运行

二进制的守护进程
-----------------
 * 使用C语言编写，高效且内存占用少
 * 便携 

为复杂环境而生
-----------------
 * 使用Zabbix Proxy 可以轻松地实现远程监视
